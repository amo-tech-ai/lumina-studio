## 📱 Responsive / UX Considerations

* **Desktop:** Two-column layouts where appropriate, hover states on cards, large hero media, grid galleries.
* **Tablet:** Collapse two-columns into one where needed, reduce grid to two cards per row, ensure readability.
* **Mobile:** Single column flow, hero image full width, service cards stacked, large tappable buttons, optimized form.
* Use lazy-loading for video thumbnails/images, ensure autoplay/muted video in hero sections (if used) only if performance allows.
* Use full-width swipeable gallery or carousel for portfolio section on mobile.
* Maintain consistent spacing, accessible typography (minimum size ~16px body), high contrast for readability.

## 📐 Spacing / Responsive Guidelines

| Device            | Container Width | Section Padding Y | Grid Gap | Headline Size | Body Size |
| ----------------- | --------------- | ----------------- | -------- | ------------- | --------- |
| Desktop (≥1280px) | 1200 px         | 6–8 rem           | 2 rem    | 56–64 px      | 18–20 px  |
| Tablet (≥768px)   | 720 px          | 4–6 rem           | 1.5 rem  | 40–48 px      | 16–18 px  |
| Mobile (≤480px)   | 90 % width      | 3–4 rem           | 1 rem    | 30–36 px      | 15–16 px  |

* Maintain **12-column grid** in desktop layouts.
* Keep **max line length ~70 ch** for readability.
* CTA buttons: primary = purple (#A87DF1), secondary = white outline w/ black text.
* Image border-radius = 12 px; shadow optional for depth.
* Use scroll reveal animation (delay 100 ms per section).
* Lazy-load video thumbnails and use poster images for performance.
