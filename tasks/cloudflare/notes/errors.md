# Can `git-ai` use AWS Bedrock?

Not directly with the package’s normal configuration.

`@git-ai/cli` v1.0.6 documents these settings:

```bash
git ai set-baseURL <OpenAI-compatible-base-url>
git ai set-key <api-key>
git ai set-model <model>
```

It expects an API-key-based, OpenAI-style HTTP service. Amazon Bedrock normally uses AWS credentials and SigV4 authentication, not a simple static OpenAI API key.

You could use Bedrock only through an intermediary that exposes an OpenAI-compatible endpoint and handles AWS authentication, but that adds unnecessary complexity for commit messages.

## Best recommendation

For this repository:

```text
Normal git commit/push for production work
PR-Agent on Bedrock for automated PR review
No git-ai inside Claude pre/post hooks
```

`git-ai` is convenient for optional commit-message drafting, but its automatic `fetch → merge → push` behavior is risky in a repo with many worktrees and parallel branches.
